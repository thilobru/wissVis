Detailed Description

base class of data algorithms

It represents an algorithm that only can output data objects but cannot create any visual output.

Note
If you want to output graphics: Implement a VisDataAlgorithm.
For visual debugging of the computation progress, please use export a DataObject that describes your debugging data and use default visualization algorithms for analysis of that data. We have, e.g., LineSet for Point and Line-Data, Grid for meshes or TensorField for scalar or vector field data.

Base class for algorithm specific options.

You can rely on the Algorithm::Options class being instantiated exactly once per algorithm instance (instantiation of AlgorithmImpl). Its life time corresponds to the life time of the algorithm instance. Thus, it is explicitly allowed to define member functions (in the Options object) and use them as option callbacks via std::bind.
