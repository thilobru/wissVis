Note
To be precise and brief, the reference documentation uses some simple conventions: Not all exceptions of a member are given in its documentation, e.g. many may result in an std::bad_alloc on insufficient memory. The containing class often states the general rules for when exceptions are thrown and additional exceptions thrown are documented at the member. The documentation on thrown exceptions also include an exception guarantee (basic, strong). Also the thread-safety statements are usually done in summary at the detailed class description. "Re-entrant" means that the members may not be used concurrently for the same object, but can be used concurrently on different objects (cf. http://doc.qt.io/qt-5/threads-reentrancy.html)
