Detailed Description

Event based input for interactions.

Events are connected here and processed by the processEvent method. Results of the event processing are stored in a state object. Thereby, the state type must satisfy the following requirements:

  • it must be serializable, so that the state that is created by interactions can be stored to the session file.
  • The type must be able to be assigned to itself, so that the state can be copied into the executing thread of the algorithm.
  • The type must contain a member function processEvent(const Event&) to act on incoming events.
