Detailed Description

Event handler for drop events.

The behavior follows that of QT's drag and drop: Whenever a drag enters the widget, dragEnterEvent is called and the event carries information about the dragged items. To receive further events, the event has to be accepted.

If the event has been accepted in the dragEnterEvent, on every move, a dragMoveEvent is raised. The event has to be accepted to receive drop events.

If the last dragMoveEvent has been accepted, a drop event is raised upon the release of the mouse and the application should perform the appropriate actions. In this case, a better checking of the event should take place and the event should only be accepted when the drop event has been completed successfully.
