Detailed Description

FAntoM provides its own set of mathematical data structures. The core data structure is a tensor, which is a generalization of scalars, vectors, and matrices. A tensor may be defined on any body of numbers including complex values via its first template parameter. A tensor has an order (0 - scalar, 1 - vector, 2 - matrix) and dimensions, which are specified using variadic templates. The order of the tensor is automatically inferred from the number of template parameters. FAnToM does not (yet) support efficient storage for symmetric or diagonal tensors.

The list above omits a few mathematical operations. Operators like + and - operate element-wise and multiplication with scalars and other elements performs according to common mathematical use, e.g. multiplication of two matrices performs matrix multiplication rather than per-element multiplication. Multiplication of two vectors performs their inner product.

All these data structures are scope-allocated, which means that they never use new to allocate data. As such they are fast, slim, can be put into containers without per-element overhead, and operations on them not throw and are re-entrant. However, you should not use large dimensions with these data structures and selection of dimensions at run-time is not possible.
