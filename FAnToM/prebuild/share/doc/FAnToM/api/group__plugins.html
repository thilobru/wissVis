Detailed Description

FAnToM can be extended by writing plugins. A FAnToM plugin is a shared object/dynamically linked library that contains one or more algorithms.

This part of the tutorial assumes that FAnToM is installed system-wide or into a path inside the user's PATH. It also assumes that CMake 2.6.3+ is installed.

To create a new plugin, FAnToM provides a convinience script that sets up a skeleton for the algorithm definition and a basic environment to build the plugin. From a shell, issue

$ fantom-makeplugin HelloWorld

It will create a directory called HelloWorld which contains a CMakeLists.txt that sets the correct paths to FAnToM's headers and libraries as well as a HelloWorld.cpp which contains the skeleton for the algorithm definition HelloWorldAlgorithm . Using

$ mkdir HelloWorld-build
$ cd HelloWorld-build
$ cmake ../HelloWorld
$ make

you configure and make the plugin. Finally you can run fantom and tell it to use the plugin by

$ fantom -p ./libHelloWorld.so

under Linux and OSX and

$ fantom -p ./HelloWorld.dll

under Windows.

You can then start the HelloWorld algorithm as usual by selecting it from the "Algorithm" menu and running it. It should produce a small message box containing "Hello World".

In general, you can provide a different name to the fantom-makeplugin script and the plugin will be named accordingly. A plugin may contain multiple algorithm definitions and their code may be distributed among several cpp files. The generated CMakeLists.txt adapts gracefully to this situation, you only have to re-run "cmake" each time after adding files to the source directory.

Each algorithm definition must be accompanied by a register object. In the HelloWorld example it was:

AlgorithmRegister<HelloWorldAlgorithm>
dummy("HelloWorld", "executes HelloWorld");

which creates a special object that automatically registers the algorithm definition for use by FAnToM when the plugin loads and unregisters it when the plugin unloads. The two parameters give the name of the algorithm and a short descriptive info about it, respectively. The name may be changed arbitrarily, but if multiple algorithms with the same name exist, it is undefined which one is executed. You may use the separator "/" in the name to organize algorithms into submenus. FAnToM's core plugins typically use "plugin name/algorithm name".

If you developed multiple separate plugins you can put them in a common directory and then batch-load them using

$ fantom -P /path/to/plugin-dir
