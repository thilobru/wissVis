Detailed Description

base class of options

Subclasses can add inputs with the various add(...) methods during constructor. The input data can then be accessed with the set- and get-methods.

Exceptions
nothingis thrown by the destructor.
std::bad_allocis thrown on insufficient memory (strong).
Note
All reading members are re-entrant.

This class follows the private implementation design pattern. The implementation is kept invisible from the user of the options (i.e., derived classes). The abstract interface to the implementation can be found in fantom::Options::Control.
