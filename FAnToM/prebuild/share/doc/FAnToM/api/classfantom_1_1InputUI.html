Detailed Description

base class for UI components connected to Inputs

Subclasses need to be registered. A static method const RTTI::TypeId& getInputType() needs to be provided. It should return the TypeId of the most general Input this InputUI can handle and is used to find the correct matching InputUI-type for a given Input-type. New InputUI-Types should be registered with an InputUIRegister.
