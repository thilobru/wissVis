Detailed Description

abstract base class of algorithms

It represents the persistent part of a computation, i.e., data members can be used to store intermediate results for accelerating computations. This base class is not intended for direct use! There are specialized algorithm types data processors (DataAlgorithm) and visualizations (VisAlgorithm). The execute-function must be strongly exception safe!

Exceptions
nothingis thrown by the destructor, debugLog(), infoLog(), warningLog().
std::bad_allocis thrown on insufficient memory (strong).
Note
The members debugLog(), infoLog(), warningLog() return different output streams. The output streams of different algorithms are also different. Content of a stream is only shown on std::flush and std::endl. The content may be formated and written to console, file, and/or screen.
All members and access to inputs as well as outputs is thread-safe. Usually, the algorithm can presume that is is executed in the only thread of the system. In case of visual outputs, however, the returned objects may live within a different thread. Particularly for 2D scenes and GUI windows, an own synchronization is mandatory!
See also
This class follows the private implementation design pattern. The implementation is kept invisible from the algorithm side. The abstract interface to the implementation can be found in fantom::AlgorithmImpl, its implementation of the encompassed functionality is found in the Control classes of Options, DataOutputs and VisOutputs, as well as in the fantom::kernel namespace.

◆ Options

using fantom::Algorithm::Options = fantom::Options

Base class for algorithm specific options.

You can rely on the Algorithm::Options class being instantiated exactly once per algorithm instance (instantiation of AlgorithmImpl). Its life time corresponds to the life time of the algorithm instance. Thus, it is explicitly allowed to define member functions (in the Options object) and use them as option callbacks via std::bind.

Performs the computation of this algorithm.

The implementation has to assure to release all existing outputs, it does not need. During execution, abortFlag might become true. In that case, the execution shall be canceled without an exception, i.e., the function should simply return.
