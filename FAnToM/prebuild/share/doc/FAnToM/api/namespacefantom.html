Detailed Description

FAnToM API namespace.

This file defines the basic topological information of the cell types used in FAnToM.

The basic structure is:

struct NewCellType
{
static const Cell::Stats stats; //< the stats object
static const Cell::Type type = Cell::NEWCELLTYPE; //< the unique name of the cell
static const std::size_t dimension = 0; //< the topological dimension of the cell
static const std::size_t numPoints = 1; //< the number of vertices in the cell
static const std::size_t numValues = numPoints; //< the number of values in a cell
static const std::size_t numFaces = 0; //< the number of faces of a cell
};

Every cell can contain an interpolate function that computes the interpolation weights for the cell vertices form the internal coordinates:

void interpolate( double coefficients[], const double coordinates[] );

In addition, a cell may implement contain-functions for any spatial dimension where it will be used:

bool contains( double coordinates[], const PointD& position, const ValueArray<VectorD>& points, const
*std::size_t indices[] );

It returns true iff the cell contains the position and in that case returns the internal coordinates in coordinates.

◆ makeLookup()

std::shared_ptr< std::vector< size_t > > fantom::makeLookup ( std::vector< size_t > &  indices)

Create a lookup into a parent domain.

Determines the number of unique entries in indices (N). Returns a lookup from [0, N-1] to the corresponding indices in the parent list. Further, all parent indices in indices are replaced by the new child indices.

indices does not need to be duplicate free. Therefore, it is possible to pass the index list of a cell complex into this function. The vertex indices corresponding to indices of the parent domain will then be converted to vertex indices corresponding to the equivalent indices of the sub domain.
