Detailed Description

base class of visualization algorithms

It represents an algorithm that cannot have any data output but only created visual output. Private data members can be used for caching to accelerate the interactive visualization.

Note
By default, VisAlgorithms use interactive options, i.e., they automatically re-execute when an option changes. To override this behavior, re-declare the static member isAutoRun with a false value.
If you want to output data: Implement a DataAlgorithm.

Base class for algorithm specific options.

You can rely on the Algorithm::Options class being instantiated exactly once per algorithm instance (instantiation of AlgorithmImpl). Its life time corresponds to the life time of the algorithm instance. Thus, it is explicitly allowed to define member functions (in the Options object) and use them as option callbacks via std::bind.
