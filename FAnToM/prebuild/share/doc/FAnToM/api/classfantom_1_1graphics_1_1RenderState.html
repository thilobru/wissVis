Resets the currently bound render target to the given color and depth

Note
The RenderState is the most logical choice to do the clearing, as the RenderTarget hat to be bound to perform a Clear. Currently, only one background color is supported. It could be handy to support multiple background colors for multi-target FrameBuffers. It would make sense to then move the color definition to the RenderTarget instead.
